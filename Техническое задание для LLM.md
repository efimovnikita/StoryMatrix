# **ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ПРОМПТ ДЛЯ РАЗРАБОТКИ)**

**Роль:** Ты — Senior Frontend Developer (React/Vite). Твоя задача — написать PWA-приложение для изучения языков по предоставленному ТЗ и мокапу интерфейса.  
**Цель:** Создать полностью рабочее клиентское приложение (Client-Side Only) без бэкенда, которое взаимодействует с Mistral AI API напрямую из браузера.

## **1\. Стек технологий**

* **Core:** React 18, Vite.  
* **Styling:** Tailwind CSS, lucide-react (для иконок).  
* **State Management:** React Hooks (useState, useEffect, useCallback), кастомные хуки для работы с localStorage.  
* **API:** fetch для прямых запросов к https://api.mistral.ai/v1/chat/completions.

## **2\. Архитектура и хранение данных (LocalStorage)**

Приложение должно сохранять свое состояние при перезагрузке страницы.  
Необходима структура в localStorage (ключ storyMatrixState):  
{  
  "settings": {  
    "mistralApiKey": "строка (запрашивать при первом входе)",  
    "targetLanguage": "it",  
    "nativeLanguage": "ru"  
  },  
  "session": {  
    "globalDirection": "строка (общий промпт)",  
    "localPrompts": { "connectors": "", "subjects": "", "verbs": "", "prepositions": "", "nouns": "", "adjectives": "" },  
    "storyLog": \["Массив сохраненных предложений"\],  
    "currentMatrix": { "Объект с текущими словами на доске (см. структуру ниже)" }  
  }  
}

## **3\. Требования к UI/UX (Компонентный подход)**

Я предоставляю тебе готовый мокап в одном файле. Твоя задача — разбить его на логические компоненты и оживить логику.  
Ожидаемая структура компонентов:

* App.jsx — точка входа, управление стейтом и проверка API ключа.  
* components/SettingsModal.jsx — модалка для ввода Mistral API ключа при старте.  
* components/StoryView.jsx — отображение истории (storyLog).  
* components/GlobalPrompt.jsx — инпут общего вектора сюжета и кнопка "Обновить все".  
* components/MatrixSection.jsx — переиспользуемый компонент для каждой из 6 категорий слов.  
* components/WordButton.jsx — кнопка отдельного слова (с поддержкой перевода).  
* components/SmartFooter.jsx — нижняя панель (гибридный ввод: блоки/текст, логика проверки грамматики).

**Ключевые фичи интерфейса (реализовать как в мокапе):**

1. Гибридный ввод: возможность собирать фразу кликами по словам ИЛИ переключиться в режим textarea и вводить текст/править окончания руками.  
2. Независимая загрузка (isLoading) для каждой секции матрицы при точечном реролле.  
3. Состояния смарт-футера: building (сборка), checking (анимация проверки LLM), result (показ карточки с ошибкой/успехом).

## **4\. Интеграция с Mistral AI API (Модуль api.js)**

Выдели всю работу с Mistral API в отдельный сервис (например, src/services/mistralApi.js).  
Используй модель mistral-large-latest и обязательно включай параметр response\_format: { type: "json\_object" }.

### **4.1 Метод: Генерация/Обновление матрицы (generateMatrix)**

**Функция:** Принимает storyLog, globalDirection, localPrompts и список секций для обновления (массив ключей, например \['nouns', 'verbs'\]).  
**Системный промпт для Mistral:**  
"Ты — генератор словаря для изучения итальянского. Проанализируй историю пользователя и вектор сюжета. Выдай новые слова для запрошенных частей речи.  
ПРАВИЛА:

1. Перевод на русский (ru) строго 1-2 слова.  
2. Глаголы (verbs) возвращать в 3 временах (past, present, future). Лицо выбирай исходя из логики истории (по умолчанию 1-е лицо ед.ч. 'Io', но если контекст требует другого — адаптируй).  
3. Верни строгий JSON."

**Пример ожидаемого JSON-ответа (только для запрошенных ключей):**  
{  
  "nouns": \[  
    { "id": "n1", "it": "il corridoio", "ru": "коридор" }  
  \],  
  "verbs": \[  
    {  
      "group": "бежать",  
      "past": { "id": "v1\_p", "it": "sono scappato", "ru": "убежал" },  
      "present": { "id": "v1\_pr", "it": "scappo", "ru": "убегаю" },  
      "future": { "id": "v1\_f", "it": "scapperò", "ru": "убегу" }  
    }  
  \]  
}
